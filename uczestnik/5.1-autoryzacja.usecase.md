# Przypadki testowe — 5.1 Autoryzacja

**Data:** 2026-07-13  
**Sekcja PRD:** 5.1 Autoryzacja  
**Pokryte wymagania:** REG-01, REG-02, REG-03, REG-04, REG-05, REG-06, REG-07, REG-08  
**Status:** Zatwierdzone

---

## Pokryte wymagania

| ID    | Opis                                                                        |
|-------|-----------------------------------------------------------------------------|
| REG-01| Użytkownik może zarejestrować konto podając: imię, nazwisko, e-mail, hasło  |
| REG-02| Walidacja pól rejestracji: e-mail (RFC 5321, unikalność), hasło (8 znaków, cyfra, znak specjalny), wymagane pola |
| REG-03| Po rejestracji system wysyła e-mail weryfikacyjny; konto aktywne po kliknięciu linku |
| REG-04| Użytkownik może zalogować się e-mailem i hasłem |
| REG-05| Po 5 nieudanych próbach logowania konto zostaje zablokowane na 15 minut |
| REG-06| Użytkownik może zresetować hasło przez e-mail (link ważny 1 godzinę) |
| REG-07| Sesja wygasa po 30 minutach bezczynności; przed wygaśnięciem wyświetlane ostrzeżenie |
| REG-08| Wylogowanie usuwa dane sesji po stronie serwera |

## Przypadki testowe

### TC-REG-01 — Pomyślna rejestracja z poprawnymi danymi

| Pole           | Wartość                                                        |
|----------------|----------------------------------------------------------------|
| **ID**         | TC-REG-01                                                      |
| **Priorytet**  | Critical                                                       |
| **Typ**        | Functional                                                     |
| **Powiązanie** | REG-01                                                         |

**Warunki wstępne:**
- Aplikacja dostępna na ekranie rejestracji
- E-mail testowy nie istnieje w systemie

**Dane testowe:**

| Zmienna            | Wartość              |
|-------------------|----------------------|
| imię              | Jan                  |
| nazwisko          | Kowalski             |
| e-mail            | jan.kowalski@test.pl |
| hasło             | MyPass123!           |
| potwierdzenie     | MyPass123!           |

**Kroki testowe:**

| # | Akcja                                                  | Oczekiwany rezultat                                        |
|---|--------------------------------------------------------|------------------------------------------------------------|
| 1 | Wejdź na stronę rejestracji                            | Formularz rejestracji wyświetlony                          |
| 2 | Wypełnij pole "Imię": "Jan"                            | Pole akceptuje wartość                                    |
| 3 | Wypełnij pole "Nazwisko": "Kowalski"                   | Pole akceptuje wartość                                    |
| 4 | Wypełnij pole "E-mail": "jan.kowalski@test.pl"         | Pole akceptuje wartość; brak błędu walidacji              |
| 5 | Wypełnij pole "Hasło": "MyPass123!"                    | Pole akceptuje wartość; wskaźnik siły hasła wyświetlony   |
| 6 | Wypełnij pole "Potwierdzenie hasła": "MyPass123!"      | Pole akceptuje wartość                                    |
| 7 | Kliknij przycisk "Zarejestruj"                         | Konto utworzone; wyświetlony komunikat potwierdzenia       |
| 8 | Sprawdź skrzynkę e-mail testowego                      | E-mail weryfikacyjny odebrany z linkiem aktywacyjnym      |

**Notatki:** Test obejmuje happy-path rejestracji. Wymaga wirtualnej skrzynki e-mail do potwierdzenia.

### TC-REG-02 — Rejestracja z brakującymi polami

| Pole           | Wartość               |
|----------------|------------------------|
| **ID**         | TC-REG-02             |
| **Priorytet**  | High                  |
| **Typ**        | Negative              |
| **Powiązanie** | REG-02                |

**Warunki wstępne:**
- Aplikacja dostępna na ekranie rejestracji
- Wszystkie pola formularza puste

**Dane testowe:**

| Zmienna            | Wartość           |
|-------------------|-------------------|
| imię              | (puste)           |
| nazwisko          | Kowalski          |
| e-mail            | (puste)           |
| hasło             | MyPass123!        |
| potwierdzenie     | MyPass123!        |

**Kroki testowe:**

| # | Akcja                                                       | Oczekiwany rezultat                                             |
|---|-------------------------------------------------------------|-----------------------------------------------------------------|
| 1 | Wejdź na stronę rejestracji                                 | Formularz rejestracji wyświetlony                               |
| 2 | Pomiń pole "Imię", wypełnij pozostałe pola poprawnie        | Pola akceptują wartości                                         |
| 3 | Kliknij przycisk "Zarejestruj"                              | Komunikat błędu pod polem "Imię": "To pole jest wymagane"       |
| 4 | Wróć, wypełnij "Imię", pomiń pole "E-mail"                 | Pola akceptują wartości                                         |
| 5 | Kliknij przycisk "Zarejestruj"                              | Komunikat błędu pod polem "E-mail": "To pole jest wymagane"     |
| 6 | Spróbuj wysłać formularz z pustymi wszystkimi polami        | Komunikaty błędów wyświetlone przy wszystkich wymaganych polach |

**Notatki:** Test walidacji pól obowiązkowych. Komunikaty błędów powinny być wyświetlane inline.

---

### TC-REG-03 — Walidacja e-mail: format błędny

| Pole           | Wartość               |
|----------------|------------------------|
| **ID**         | TC-REG-03             |
| **Priorytet**  | High                  |
| **Typ**        | Negative              |
| **Powiązanie** | REG-02                |

**Warunki wstępne:**
- Aplikacja dostępna na ekranie rejestracji
- Pozostałe pola formularza wypełnione poprawnymi danymi

**Dane testowe:**

| Zmienna            | Wartość              |
|-------------------|----------------------|
| imię              | Jan                  |
| nazwisko          | Kowalski             |
| e-mail            | invalid.email.pl     |
| hasło             | MyPass123!           |
| potwierdzenie     | MyPass123!           |

**Kroki testowe:**

| # | Akcja                                                      | Oczekiwany rezultat                                     |
|---|------------------------------------------------------------|---------------------------------------------------------|
| 1 | Wejdź na stronę rejestracji                                | Formularz rejestracji wyświetlony                       |
| 2 | Wypełnij wszystkie pola, ale e-mail: "invalid.email.pl"   | Pola akceptują wartości; brak znaku "@"                 |
| 3 | Kliknij przycisk "Zarejestruj"                             | Komunikat błędu: "E-mail ma nieprawidłowy format"       |
| 4 | Zmień e-mail na: "test@"                                  | Pole akceptuje wartość                                  |
| 5 | Kliknij przycisk "Zarejestruj"                             | Komunikat błędu: "E-mail ma nieprawidłowy format"       |
| 6 | Zmień e-mail na: "test @example.com" (spacja)             | Pole akceptuje wartość                                  |
| 7 | Kliknij przycisk "Zarejestruj"                             | Komunikat błędu: "E-mail ma nieprawidłowy format"       |

**Notatki:** Test walidacji formatu e-maila. Obejmuje wiele negatywnych wariantów formatu.
